# Change Log
Все изменения этого проекта будут задокументированы в этом файле.

## [0.0.4] - 31.01.2023
### Добавлено:
* SessionAreaBorders. Сущность для определения границ игрового пространства. Вычисляет текущую границу перед кораблем. 
* ICameraView. Интерфейс, от которого можно будет наследовать различного рода камеры в игре. Предоставляет множество методов: слежение за обьектом, вращение, нахождение рандомной точки в обзоре, проверка видимости обьекта камерой, включение и выключение. Так же хранит позицию и жесткость слежения и вращения. 
  * IShipCameraView. Интерфейс для основной камеры, которая будет следовать за кораблем.
  * MVP ShipCamera связка и updater. Используется для просчета текущей жесткости и ее изменения, в зависимости от скорости.
* SessionSetupUtilitiesPresenter. Presenter, отвечающий за инициализацию камеры и границ.

### Изменено:
* SessionSceneView. Добавлен метод для настройки камеры, а так же поле для получения IShipCameraView.
* SessionPresenter. Добавлена инициализация SessionSetupUtilitiesPresenter.
* SessionLocationGameModel. Добавлены необходимые интерфейсы.

## [0.0.4] - 30.01.2023
### Добавлено:
* MVP Ship связка. 
  * ShipModel. Хранит спецификацию, текущую позицию, скорость, ускорение и здоровье. Так же может принимать урон и сообщать об этом через ивент.
  * ShipPresenter. Presenter, запускающий все необходимые presenter'ы и updater'ы: ShipDamagePresenter, ShipBoostEffectPresenter, ShipPhysicsUpdater.
    * ShipDamagePresenter. Отвечает за получение урона.
    * ShipBoostEffectPresenter. Отвечает за отображение эффекта ускорения корабля, в зависимости от текущих скорости и ускорения.
  * ShipView. Предоставляет методы для получения и установки позиции и скорости, управления эффектами ускорения. 
* ShipPhysicsUpdater. Updater для просчета физики полета корабля, в зависимости от ввода игрока.
* SessionSetupShipPresenter. Presenter, отвечающий за создание объекта корабля и инициализацию MVP Ship связки. 
* IEntity. Интерфейс для всех будущих entity в игре. Предоставляет поле для текущего здоровья.
* IDamageable. Интерфейс для сущностей, имеющих возможность принимать урон. Предоставляет метод для получения урона.
* IMovable. Интерфейс для сущностей, имеющих возможность передвигаться. Предоставляет поле для текущей позиции.

### Изменено:
* SessionSceneView. Добавлены методы для создания корабля и input'a, а так же поле для получения IShipView.
* SessionPresenter. Добавлена инициализация SessionSetupShipPresenter.
* GameModel. Добавлен интерфейс IShipModel.

## [0.0.2] - 24.01.2023
### Добавлено:
* AddressableSceneLoadWrapper. Реализация абстрактного класса LoadSceneWrapper. В методе Load создает связку Model-Presenter. Далее вызывается метод Init у Presenter'a, который производит загрузку сцены с помощью Addressables. По окончании загрузки, создается LocationLogicBuilder, инициализирующий верхнеуровневую связку Model-Presenter конкретной локации.
* AddressableObjectLoadWrapper. Реализация интерфейса ILoadObjectWrapper. Работает аналогично wrapper'у сцен, однако по окончании загрузки объекта сохраняет T результат и завершает ожидание.
* LocationLogicBuilder. Содержит метод Build, при вызове которого на новой сцене находится игровой объект с именем из спецификации сцены и создается верхнеуровневая связка Model-Presenter конкретной локации по ключу сцены.
* LocationSceneView. Сущность, наследующаяся от MonoBehaviour. От нее будут наследованы все будущие верхнеуровневые View локаций.

### Изменено:
* Startup. Написаны необходимые инициализации основных сущностей.
* GameModel. Добавлены некоторые интерфейсы основных сущностей.  
* Assembly Definition. Добавлена зависимость Unity.ResourceManager.

## [0.0.2] - 23.01.2023
### Добавлено:
* GameSpecifications. Сущность, которая хранит все спецификации в формате INewSpecificationsCollection. При создании вызывается метод Load, в котором ожидается загрузка каждого типа спецификаций через LoadSpecificationsWrapper по ключу.
* LoadSpecificationsWrapper. Сущность, использующаяся для загрузки спецификации. По Startup спецификации выбирает то, каким способом будет происходить загрузка (Json или ScriptableObject).
* NewSpecificationsCollection\<T>. Сущность, хранящая Dictionary<string, T> спецификации определенного типа, где ключ - Id в спецификации.
* Ключи коллекций спецификаций в Addressables для Json и ScriptableObject файлов.

## [0.0.2] - 21.01.2023
### Добавлено:
* Unity package Input System.
* ShipInputAsset. Input Action Asset, содержащий необходимые бинды для управления космическим кораблем.
  * Thrust [W\S] - движение вперед\назад.
  * Strafe [A\D] - движение влево\вправо.
  * UpDown [Ctrl\Space] - движение вниз\вверх.

  * PitchYaw [Mouse delta] - подъем\опускание носа.
  * Boost [LShift] - ускорение.
  * MousePosition [Mouse position] - позиция мыши на экране.

## [0.0.2] - 20.01.2023
### Добавлено:
* Основные будущие сцены для игры.
  * CoreScene - сцена, с которой запускается игра. На ней присутствует скрипт Startup, инициализирующий все необходимые механики.
  * HomeScene - сцена, которая будет автоматически запускаться после прогрузки CoreScene. Будет содержать интерфейс с кнопками настройки, выхода, быстрого запуска сессии и входа в док-локацию.
  * DockScene - сцена, которая будет содержать предпоказ кораблей, их характеристики и цены, количество монет игрока и кнопку игры. На этой сцене игрок сможет контролировать свой инвентарь и ресурсы.
  * SessionScene - сцена сессии игры.
* Startup - скрипт, инициализирующий все необходимое для корректной работы игры.
* GameModel - реализация интерфейса IBaseGameModel, где будут содержаться все основные модели игры.
